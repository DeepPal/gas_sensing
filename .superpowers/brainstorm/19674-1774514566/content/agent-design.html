Design Section 3 — Agentic AI Architecture (Hybrid)

The novel core of SpectraAgent. Two clearly separated layers — deterministic signal math + Claude API reasoning.

Two-Layer Agent Architecture
DETERMINISTIC AGENTS always running · <1 ms · no API cost
  • QualityAgent SNR gate, saturation check, outlier flagging. Hard-blocks on saturation only. Emits quality events per frame.
  • DriftAgent Peak shift trend monitoring via CUSUM. Emits WARN when drift rate > threshold. Stores drift history.
  • CalibrationAgent AIC model selection across GPR/Langmuir/Freundlich/Hill. Triggers retraining when R² drops below threshold.
  • ExperimentPlannerAgent Bayesian optimization (BoTorch) to suggest next concentration. Maximises information gain in calibration space.
CLAUDE API AGENTS on-demand · async · never in signal path
  • AnomalyExplainer Given a drift alert + recent data → explains in plain English: "This pattern is consistent with surface fouling. The blue-shift rate of 0.08 nm/hr exceeds typical thermal drift."
  • ExperimentNarrator Explains WHY the Bayesian planner suggested a concentration: "The 0.05 ppm point would halve your LOD uncertainty. Your current gap between 0.1 and 0.01 ppm is statistically undersampled."
  • ReportWriter Given session metadata + results → writes Methods + Results sections in journal style (Sensors and Actuators B format). Fills in your actual numbers.
  • DiagnosticsAgent On hardware error or unexpected spectrum shape → explains likely cause and suggests fix: "VI_ERROR_TMO typically means a stale VISA handle — try disconnecting and reconnecting."
