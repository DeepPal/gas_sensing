3 ways to build the full web app

All three replace Streamlit with FastAPI. The difference is the frontend technology stack.

A

FastAPI + Jinja2 + HTMX + uPlot ★ Recommended

Server renders HTML pages via Jinja2 templates. HTMX handles partial page updates without full reloads (e.g. clicking "Run Preprocessing" swaps just the results panel). uPlot handles all charts. No npm, no build step — pure Python + HTML files.

Stack: FastAPI · Jinja2 · HTMX · Alpine.js · uPlot
Packaging: Python wheel → pip install lspr-tool && lspr-tool start
Handoff: A lab member who knows Python can read and edit it

Pros

  • No JS build toolchain
  • Easiest to hand off
  • Builds directly on live_server.py
  • 20 Hz WebSocket charts

Cons

  • Less interactive than React
  • Templates are split across files
B

FastAPI + React (SPA)

Separate frontend (React/Vite) and backend (FastAPI). React components for each tab, REST API for data, WebSocket for live charts. The kind of stack used by commercial instrument software like PicoScope or Ocean Optics OceanView.

Stack: FastAPI · React · Vite · TypeScript · Recharts/uPlot
Packaging: Docker container or bundled with PyInstaller
Handoff: Requires someone comfortable with JS/React

Pros

  • Richest UI components
  • Best type safety
  • Most "modern" stack

Cons

  • npm + Vite toolchain
  • Two separate codebases
  • Harder to hand off to physicist
C

FastAPI + Vanilla HTML/JS (extend live_server.py)

Directly extend the existing live_server.py pattern — one large HTML string per page, served from FastAPI routes. Works today for the Live tab; repeat the pattern for all tabs. No framework overhead at all.

Stack: FastAPI · Plain HTML/CSS/JS · uPlot
Packaging: Python wheel
Handoff: Easy to read, hard to maintain as it grows

Pros

  • Zero new dependencies
  • Fastest to start

Cons

  • HTML-in-Python strings don't scale
  • No component reuse
  • Becomes unmaintainable past ~3 pages

A is recommended: no build toolchain, Python-native, directly extends the existing live_server.py foundation, and a physics PhD can maintain it
