Who uses this tool, and how?

This determines packaging, deployment model, and multi-user needs

A

Just you — one PC, one spectrometer

You run it on the lab PC directly connected to the CCS200. No one else touches it. "Deployable" means you want a clean one-command start, not a YAML file maze — not a shared server.

B

Small lab team — 2–5 people, each running their own instance

Lab members each install and run it locally on their own machines or the shared lab PC. They're comfortable with Python but not necessarily with config files. You need a clean installer and sensible defaults.

C

Shared lab server — members connect via browser

One machine runs the software, lab members access it from laptops via a local network URL. Central session history, shared calibration models, single point of maintenance.

D

Mixed — mostly you now, but designed to hand off later

You're the primary user today, but the goal is a tool that a new lab member could pick up in a week with no hand-holding. Architecture should support this, even if multi-user features come later.
